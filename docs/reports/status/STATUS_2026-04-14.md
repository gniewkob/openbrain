## Kluczowe naprawy od v2.2 (2026-03-28 → 2026-04-14)

### Bezpieczeństwo
- `hmac.compare_digest` dla `X-Internal-Key` (timing-safe)
- `X-Request-ID` sanityzacja (log injection prevention)
- Thread-safe `POLICY_REGISTRY` (atomic swap zamiast clear+update)
- `content` redacted z INFO-level request logów

### Poprawność danych
- JSONB mutation tracking (SQLAlchemy `MutableDict` — `title`, `custom_fields` nie były persistowane)
- `match_key` wymagany dla corporate domain (zapobiega un-dedupable duplicates)
- Cross-domain dedup fix (GROUP BY dodano `domain`)
- `_requires_append_only` bug — błędnie blokował `build` domain Decision records
- N+1 query fix w `store_memories_bulk` (batch fetch `IN (ids)`)
- `sensitivity` change detection (był pomijany w `changed` check)
- Embedding fetch przed mutacją pól (zapobiegał partial write)

### Governance
- `dry_run=True` nie zapisuje już AuditLog
- Corporate `upsert` auto-upgrade do `append_version`
- `updated_by` — authenticated actor zamiast request-level override
- `MemoryUpdate.content` opcjonalne (metadata-only patches bez resupply)

### Obsidian
- Vault discovery fallback do `OBSIDIAN_VAULT_PATHS` env gdy CLI niedostępne
- Direct filesystem read via `aiofiles` (bypass CLI dla szybkości)
- Bidirectional sync engine z double-check locking
